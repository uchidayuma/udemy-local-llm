# データの準備 STEP3：メタデータ付与

**YAMLフロントマターによるタグ・数値・対象ユーザーの構造化**

---

## このステップの目的

STEP2までで「人間が読みやすい文書」は完成しています。
STEP3では、**機械が検索・フィルタリングしやすいメタデータ**をファイルの先頭に追加します。

これをYAMLフロントマター（YAML front matter）と呼びます。

---

## YAMLフロントマターとは

ファイルの先頭に `---` で囲んで書く、キーと値のペアです。

```
---          ← 開始の区切り線
キー: 値
キー: 値
---          ← 終了の区切り線

# 本文がここから始まる
```

AnythingLLMはこのメタデータを文書の属性として認識し、
テキスト検索とは別の軸でフィルタリングや優先度付けに活用します。

---

## 基本テンプレート

打田電器の製品仕様書に使用する標準テンプレートです。

```yaml
---
model_id: UD-1200XP
product_name: プレミアムペットケア UD-1200XP
brand: 打田電器
price_tier: プレミアム
tags: [プレミアム, ペット, 犬, 猫, 抜け毛, ペット消臭, 除菌, AI制御, プラズマイオン, 静音]
target_users: [ペット飼育, 犬がいる家庭, 猫がいる家庭, ペット臭が気になる, 衛生重視]
installation_depth_min_mm: 540
washing_capacity_kg: 12.0
drying_capacity_kg: 7.0
noise_wash_db: 25
---
```

---

## 各フィールドの説明

### 識別フィールド（必須）

| フィールド | 型 | 説明 | 例 |
|-----------|-----|------|----|
| `model_id` | 文字列 | 型番（半角ハイフン・大文字） | `UD-1200XP` |
| `product_name` | 文字列 | 製品の正式名称＋型番 | `プレミアムペットケア UD-1200XP` |
| `brand` | 文字列 | メーカー名 | `打田電器` |
| `price_tier` | 文字列 | 価格帯グループ | `プレミアム` / `ハイグレード` / `スタンダード` |

**price_tierの統一値：**

```
エントリー    ← Bラインの低価格帯
スタンダード  ← Sライン・Tラインの標準帯
ミドルハイ    ← Xライン入門・iライン標準
ハイグレード  ← iライン上位・Tライン上位
プレミアム    ← Xラインフラッグシップ
```

---

### 検索・フィルタリング用フィールド（必須）

#### tags（タグリスト）

ユーザーが検索に使いそうなキーワードをリストで列挙します。

```yaml
tags: [プレミアム, ペット, 犬, 猫, 抜け毛, ペット消臭, 除菌, AI制御, プラズマイオン, 静音]
```

**タグ選定の3つの基準：**

1. **ライン・グレード** → `プレミアム` `エントリー` `スリム` `IoT` `パワフル`
2. **特化機能** → `ペット` `寒冷地` `アレルギー` `省エネ` `時短` `大容量`
3. **ユーザーが検索しそうな言葉** → `犬` `猫` `北海道` `花粉症` `マンション`

> **ポイント：** タグは「仕様書に書いてある言葉」だけでなく、
> 「ユーザーが口にする言葉」も入れます。
> 例：「ヒートポンプ乾燥」と書いてある機種に `省エネ` `電気代節約` も追加する。

---

#### target_users（対象ユーザーリスト）

想定するユーザーの状況・悩みをリストで列挙します。

```yaml
target_users: [ペット飼育, 犬がいる家庭, 猫がいる家庭, ペット臭が気になる, 衛生重視, プレミアム希望]
```

**書き方のルール：**
- 短いフレーズ（3〜10文字程度）にする
- 状況・悩み・希望の3種類を混ぜる
- 型番や数値は入れない（本文に書く情報と重複するため）

```yaml
# 良い例
target_users: [4人家族, マンション, 設置スペースが狭い, 真下排水が必要, 省エネ重視]

# 悪い例（数値・型番が混入している）
target_users: [洗濯12kg対応, 防水パン540mm, UD-1200XP購入希望者]
```

---

### 数値フィールド（必須）

検索・比較・フィルタリングに使用する数値を個別フィールドとして記録します。

| フィールド | 単位 | 説明 | 例 |
|-----------|------|------|----|
| `installation_depth_min_mm` | mm | 設置に必要な防水パン奥行き最小値 | `540` |
| `washing_capacity_kg` | kg | 洗濯容量 | `12.0` |
| `drying_capacity_kg` | kg | 乾燥容量（乾燥なしモデルは `0` または省略） | `7.0` |
| `noise_wash_db` | dB | 洗い時の運転音 | `25` |

**数値フィールドのルール：**

```yaml
# 正しい書き方（数値のみ・単位なし）
installation_depth_min_mm: 540
washing_capacity_kg: 12.0
noise_wash_db: 25

# 間違った書き方（単位を含めない）
installation_depth_min_mm: 540mm      # NG
washing_capacity_kg: 12.0キログラム   # NG
noise_wash_db: 25dB                   # NG
```

> **理由：** 数値フィールドに単位を含めると文字列として扱われ、
> 「540mm以下に設置できる機種を探して」という数値比較ができなくなります。

## フィールドごとの完成例（機種別）

### Sライン（スリム）の例

```yaml
---
model_id: UD-1000S
product_name: マンションサイズ・スリム UD-1000S
brand: 打田電器
price_tier: ミドルハイ
tags: [スリム, 奥行コンパクト, マンション向け, 真下排水対応, 低振動, ヒートポンプ乾燥, 蒸気除菌]
target_users: [マンション住まい, 設置スペースが狭い, 真下排水, 3〜4人家族, 防水パンが小さい]
installation_depth_min_mm: 510
washing_capacity_kg: 10.0
drying_capacity_kg: 6.0
noise_wash_db: 28
---
```

### Xライン（プレミアム寒冷地）の例

```yaml
---
model_id: UD-1200XL
product_name: プレミアム寒冷地仕様 UD-1200XL
brand: 打田電器
price_tier: プレミアム

target_users: [北海道在住, 東北在住, 寒冷地, 凍結が心配, 厳冬期対応希望, 屋外設置]

```

## やってはいけないメタデータの書き方

### NG1：文章を書く

```yaml
# NG
tags: このモデルはペット飼育家庭に最適で犬や猫の抜け毛も安心です
```

```yaml
# OK
tags: [ペット, 犬, 猫, 抜け毛対応]
```

---

### NG2：数値フィールドに単位や「約」を入れる

```yaml
# NG
washing_capacity_kg: 約12.0kg
noise_wash_db: 25dB以下

# OK
washing_capacity_kg: 12.0
noise_wash_db: 25
```

---

### NG3：本文と重複する詳細情報をYAMLに書く

```yaml
# NG（詳細な説明はYAMLに書かない）
tags: [ペット抜け毛3段階フィルター毛取りフィルター細毛フィルター排水フィルター]

# OK（短いキーワードのみ）
tags: [ペット, 抜け毛, 3段階フィルター]
```

> **理由：** YAMLは「索引」であり「本文」ではありません。
> 詳細は `## 主な機能` セクションに書きます。

---

### NG4：フィールド名に日本語や空白を使う

```yaml
# NG
洗濯容量: 12.0
防水パン奥行き最小: 540

# OK
washing_capacity_kg: 12.0
installation_depth_min_mm: 540
```

## tags設計の参考：ライン別共通タグ

各ラインで必ず含めるべき共通タグを定義しておくと、ファイル間の一貫性が保てます。

| ライン | 必須タグ | 追加可能なタグ例 |
|--------|---------|----------------|
| **Bライン** | `エントリー`, `シンプル`, `低価格` | `高齢者向け`, `音声ガイド`, `大型文字` |
| **Sライン** | `スリム`, `奥行き600mm`, `マンション` | `真下排水`, `低振動`, `寒冷地` |
| **Tライン** | `時短`, `パワフル`, `高圧洗浄` | `泥汚れ`, `ペット`, `スポーツ`, `寒冷地` |
| **iライン** | `IoT`, `スマホ連携`, `ヒートポンプ乾燥` | `省エネ`, `UV除菌`, `スリム`, `デザイン` |
| **Xライン** | `プレミアム`, `AI制御`, `プラズマイオン` | `アレルギー`, `ペット`, `寒冷地`, `超大容量` |

## チェックリスト

### フォーマット
- [ ] ファイルの先頭が `---` で始まっている
- [ ] 終了の `---` の後に空行を1行入れてから本文を書いている
- [ ] フィールド名はすべて半角英数字・アンダースコアのみ使っている

### 識別フィールド
- [ ] `model_id` は半角ハイフン・大文字の型番になっている（例：UD-1200XP）
- [ ] `product_name` に型番と製品名の両方が含まれている
- [ ] `price_tier` は決められた5段階のいずれかになっている

### tagsフィールド
- [ ] タグはリスト形式（`[タグ1, タグ2, ...]`）で書いている
- [ ] 仕様書の用語だけでなく、ユーザーが使う言葉も含めている
- [ ] 文章ではなく短いキーワードになっている（10文字以内が目安）
- [ ] ライン共通の必須タグが含まれている

### 数値フィールド
- [ ] `installation_depth_min_mm` に数値のみ記載している（単位なし）
- [ ] `washing_capacity_kg` に数値のみ記載している
- [ ] `noise_wash_db` に数値のみ記載している

## STEP1〜3 の全体まとめ

| ステップ | 作業内容 | 効果 |
|---------|---------|------|
| **STEP1** クリーニング | ノイズ除去・表記統一・キーワード補完 | 文書の品質を上げる |
| **STEP2** 構造化 | `##` 見出しによるセクション分け | チャンク境界を明確にする |
| **STEP3** メタデータ付与 | YAMLフロントマターの追加 | 検索・フィルタリング精度を上げる |

3つのステップを経ることで、生のテキストファイルが「RAGが正確に検索できる文書」に変わります。
