# データの準備 STEP2：構造化

**見出しによるセクション分け・チャンク境界の設計**

---

## このステップの目的

RAGは文書を「チャンク」と呼ばれる小さな単位に分割してから検索します。
チャンクの分割が適切でないと、**答えが文書に存在しているのに返ってこない**という問題が起きます。

STEP2では、Markdownの見出し（`##`）を使って文書を意味のある単位に分割し、
RAGが「正しい情報の塊」を取り出しやすい構造に整えます。

---

## チャンクとは何か

AnythingLLMは文書をアップロードすると、見出し（`##`）を区切りとして文書を分割します。

```
## 基本スペック          ← チャンク1の始まり
洗濯容量：12.0キログラム
乾燥容量：7.0キログラム
                        ← チャンク1の終わり
## 設置情報             ← チャンク2の始まり
外形寸法：幅645ミリメートル × 奥行き725ミリメートル
防水パン：内寸奥行き540ミリメートル以上
                        ← チャンク2の終わり
```

ユーザーが「防水パンのサイズを教えて」と聞くと、RAGは「設置情報」チャンクを取得します。
「洗濯容量を教えて」と聞くと「基本スペック」チャンクを取得します。

---

## 1. 推奨セクション構成

以下の5セクション構成を標準テンプレートとして使用します。

```markdown
# [型番] [製品名]

## 機種概要
## 基本スペック
## 設置情報
## 主な機能
## こんなお客様に最適
```

### 各セクションの役割

| セクション | 入れる情報 | 対応する質問例 |
|-----------|-----------|--------------|
| **機種概要** | コンセプト・ターゲット・このモデルの位置づけ | 「UD-1000Sはどんな機種ですか？」 |
| **基本スペック** | 容量・乾燥方式・消費電力・年間コスト・運転音 | 「洗濯容量は何kgですか？」「年間電気代は？」 |
| **設置情報** | 外形寸法・防水パン要件・質量・排水方向 | 「防水パンの最小奥行きは？」「幅は何mmですか？」 |
| **主な機能** | 搭載機能・独自技術・スペック数値 | 「加熱蒸気除菌の温度は？」「フィルターは何段？」 |
| **こんなお客様に最適** | ユーザー像・選ぶべき理由 | 「ペットを飼っている家庭向けは？」 |

---

## 2. セクション別の書き方ルール

### 2-1. 機種概要

1〜3文の自然な日本語で書きます。型番・製品名・コンセプト・対象ユーザーを含めます。

```markdown
## 機種概要

打田電器のUD-1200XPは、ペット飼育家庭向けに特化したプレミアムモデルです。
UD-1200Xをベースに、犬・猫の抜け毛対策・ペット専用消臭システム・除菌強化を追加しました。
「プレミアムモデルの性能でペットとの暮らしを快適にしたい」という方向けです。
```

> **ポイント：** 「どんな機種か」を1チャンクで完結させます。他のセクションの情報は入れません。

---

### 2-2. 基本スペック

箇条書き（`-`）で1行1項目にします。数値の後ろに補足説明を添えます。

```markdown
## 基本スペック

- 洗濯容量：12.0キログラム
- 乾燥容量：7.0キログラム
- 乾燥方式：大風量ヒートポンプ乾燥（省エネ・衣類に優しい）
- 標準使用水量：洗濯時 75リットル／洗濯から乾燥時 52リットル
- 目安時間：洗濯 約32分／洗濯から乾燥 約147分
- 消費電力量：洗濯時 65ワット時／洗濯から乾燥時 591ワット時
- 定格電流：13アンペア
```

**年間コストは基本スペックに含める：**

```markdown
## 基本スペック（年間コスト含む）

- 年間消費電力量：約91キロワット時
- 年間電気代目安：約2,821円（31円/キロワット時換算）
- 年間水道代目安：約6,563円（0.24円/リットル換算）
- 省エネ基準達成率：123%
- 統一省エネラベル：5段階中5
```

---

### 2-3. 設置情報

寸法情報を**ミリメートル単位**で明記します。設置条件に関わる全情報を1箇所に集めます。

```markdown
## 設置情報

- 外形寸法：幅645ミリメートル × 奥行き726ミリメートル × 高さ1056ミリメートル
- 設置可能な防水パン：内寸奥行き 540ミリメートル以上
- 本体質量：約83キログラム
- 設置床荷重：150キログラム毎平方メートル以上
- 排水ホース取り出し方向：左・右・真下（3方向対応）
```

> **重要：** 防水パン寸法・外形寸法・質量は必ず同じチャンクに入れます。
> 別セクションに分けると「この機種は設置できますか？」という複合質問に答えられなくなります。

---

### 2-4. 主な機能

機能ごとに**太字の機能名**＋説明の形式で書きます。数値は必ず記載します。

```markdown
## 主な機能

- **ペット抜け毛専用3段階フィルター**：毛取りフィルター→細毛フィルター→排水フィルターの3段構成。フィルター目詰まり検知機能付き（アプリ通知）
- **ペット専用消臭プラズマ**：犬臭・猫臭・獣臭に特化したプラズマイオン配合比に調整
- **ペットコース**：毛ほぐしプレ洗浄（90秒）→高温温水本洗浄（45度）→強化すすぎ（3回）→消臭乾燥の4段階
- **洗剤自動投入（3タンク）**：液体洗剤・柔軟剤・ペット用消臭剤（最大300ミリリットル）
```

> **ポイント：** 機能名を太字にすることで、RAGが機能名で検索されたときに
> そのチャンクのスコアが上がりやすくなります。

---

### 2-5. こんなお客様に最適

箇条書きで、**具体的なユーザー像や悩み**を書きます。型番や数値は入れません。

```markdown
## こんなお客様に最適

- 犬・猫を飼っており、抜け毛の排水詰まりに長年悩んでいる方
- ペット臭を洗濯機で完全に除去したい・ペット用消臭剤の自動投入を使いたい方
- プレミアムモデルの性能でペット飼育環境を整えたい方
- 家族にアレルギーがあり、ペット毛・ペットアレルゲンの除去も重視する方
```

> **理由：** ユーザーが「ペットを飼っている家庭におすすめの機種は？」と聞いたとき、
> このセクションがマッチして正しい機種を返せます。

## 3. やってはいけない構造

### NG1：情報が1つのセクションに詰め込まれている

```markdown
## 仕様

洗濯容量12kg、乾燥7kg、幅645mm×奥行726mm×高さ1056mm、
防水パン540mm以上、年間電気代2,821円、ペット抜け毛3段階フィルター搭載、
ペットコースは毛ほぐし90秒→45度温水→すすぎ3回→消臭乾燥...
```

**問題点：** 1チャンクが大きすぎて、検索で取得されても関係のない情報が大量に混入します。

---

### NG2：セクションが細かすぎる

```markdown
## 洗濯容量
12.0キログラム

## 乾燥容量
7.0キログラム

## 乾燥方式
大風量ヒートポンプ乾燥
```

**問題点：** チャンクが小さすぎて文脈がなくなり、「この機種の洗濯容量と乾燥容量を教えて」という質問に1回の検索で答えられません。

---

### NG3：機種名がセクション内に書かれていない

```markdown
## 基本スペック

- 洗濯容量：12.0キログラム
- 乾燥容量：7.0キログラム
```

**問題点：** チャンクが切り出されたとき、どの機種の情報か分からなくなります。

**改善：** セクションの先頭か本文中に型番を含めます。

```markdown
## 基本スペック

UD-1200XPの基本性能は以下のとおりです。

- 洗濯容量：12.0キログラム
```

## 4. ファイル全体の構成例

実際の完成ファイルの骨格です。

```markdown
---
（STEP3で追加するYAMLフロントマター）
---

# UD-1200XP プレミアムペットケア

## 機種概要

（1〜3文のコンセプト説明）

## 基本スペック

- 洗濯容量：〇〇キログラム
- ...
- 年間電気代目安：約〇〇円

## 設置情報

- 外形寸法：幅〇〇ミリメートル × 奥行き〇〇ミリメートル × 高さ〇〇ミリメートル
- 設置可能な防水パン：内寸奥行き 〇〇ミリメートル以上
- ...

## 主な機能

- **機能名A**：説明（数値）
- **機能名B**：説明（数値）
- ...

## こんなお客様に最適

- ユーザー像A
- ユーザー像B
- ...
```

## チェックリスト

### セクション構成
- [ ] 5セクション（機種概要・基本スペック・設置情報・主な機能・こんなお客様に最適）を作成した
- [ ] 各セクションに `##` 見出しを付けた
- [ ] ファイル先頭に `#` で機種名・型番を記載した

### 内容の配置
- [ ] 設置寸法・防水パン要件・質量を「設置情報」に集約した
- [ ] 機能名を `**太字**` で記載した
- [ ] 各チャンク内に型番（例：UD-1200XP）が含まれている
- [ ] 年間コストを「基本スペック」に含めた

### チャンクサイズ
- [ ] 1セクションが長すぎず（目安：20行以内）、短すぎない（目安：3行以上）
- [ ] 関連する情報が同じセクションにまとまっている

## 次のステップ

構造化が完了したら、**STEP3：メタデータ付与**に進みます。
YAMLフロントマターでタグ・対象ユーザー・設置寸法などを数値・リスト形式で記述し、
フィルタリング検索に対応させます。
